FastAPI + LangChain Agent + MCP

AIOps Platform — automated detection, diagnosis and remediation of Kubernetes issues

View Reports System Health

How it works

Step 1
Event Collection
Every 3 minutes the Event Collector watches all Kubernetes Warning events across the cluster. New events are deduplicated against existing reports before being queued for analysis.
Step 2
Root Cause Analysis
The RCA Agent (LangChain + Claude Sonnet 4.5 via Requesty AI) connects to the MCP Server and uses kubectl get, describe, and logs to investigate affected resources and write a structured diagnostic report.
Step 3
Report Persistence
Each identified problem is saved as a report in PostgreSQL with severity, affected resources, root cause, evidence, and a suggested fix command. Reports are accessible via the web UI.
Step 4
Automated Remediation
For COMPLETE reports, you can trigger the Fix Agent. It reads the diagnosis, executes the suggested kubectl commands via MCP Server, verifies the result, and updates the report to FIXED or FIX_FAILED.
Stack: FastAPI (API + Web UI)  ·  LangChain + LangGraph (AI agents)  ·  MCP Server Kubernetes (kubectl tools over HTTP)  ·  Claude Sonnet 4.5 via Requesty AI (LLM)  ·  PostgreSQL (report storage)  ·  Deployed on OKE (Oracle Kubernetes Engine)
